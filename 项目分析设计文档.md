# 创新学分申领管理平台 - 项目分析设计文档

---

## 第一章 项目概述

### 1.1 项目背景

随着高校创新创业教育的深入推进，创新学分的管理与认定成为教务管理中的重要环节。传统的纸质申请、人工审核方式存在效率低下、数据统计困难、信息追溯不便等问题。为解决这些问题，本项目设计并实现了一套基于微服务架构的创新学分申领管理平台，实现学分申领的线上化、审批流程的自动化、数据统计的可视化。

### 1.2 项目目标

本项目旨在构建一个功能完善的创新学分申领管理平台，实现以下目标：

1. **学分申领线上化**：学生可通过平台在线提交学分申请，上传获奖证明材料，实时查看审批进度。
2. **审批流程自动化**：管理员可在线审批学分申请，填写审批意见，系统自动更新状态。
3. **数据统计可视化**：为管理员提供各年级获奖统计、申领状态分布、学年学分汇总等可视化图表。
4. **技术社区互动化**：搭建技术社区平台，支持学生发布求助信息、查看公告、互动回复。
5. **信息管理规范化**：集中管理比赛信息、指导教师信息，方便学生获取资讯。

## 第二章 需求分析

### 2.1 系统角色分析

本系统涉及三类用户角色：

| 角色 | 标识 | 说明 |
|------|------|------|
| 学生 | STUDENT | 系统的主要使用者，可管理个人信息、申领学分、查看比赛、发布求助等 |
| 管理员 | ADMIN | 系统的管理者，可管理比赛、审批学分、发布公告、查看统计等 |
| 指导教师 | INSTRUCTOR | 信息由管理员维护，供学生查看和联系 |

### 2.2 功能需求分析

#### 2.2.1 学生模块功能需求

| 功能项 | 需求描述 |
|--------|---------|
| 个人信息 | 学生可修改个人信息，包括头像、昵称、个人简介、联系方式等；手机号修改需跳转至安全中心 |
| 安全中心 | 学生可修改登录密码、绑定或更换手机号 |
| 比赛信息 | 学生可查看当前时间段内可参加的比赛信息，点击可直接跳转至对应的报名系统 |
| 个人获奖 | 学生可查看当前学年内个人的获奖情况及创新学分获得情况 |
| 创新学分 | 学生可通过上传获奖证明照片自主申领创新学分，选择所属学年，查看当前学分拥有情况 |
| 技术求助 | 学生可发布求助信息、查看他人求助信息、回复他人的求助帖子，实现技术交流与互助 |
| 指导教师 | 学生可查看全校指导教师的信息，包括简介、联系方式等，点击可查看详情 |

#### 2.2.2 管理员模块功能需求

| 功能项 | 需求描述 |
|--------|---------|
| 个人信息 | 管理员可修改个人信息，包括头像、昵称等 |
| 安全中心 | 管理员可修改密码、绑定手机号 |
| 比赛信息 | 管理员可对比赛信息进行添加、查看、修改、删除（CRUD）操作 |
| 学生信息 | 管理员可查看所有学生的获奖情况和创新学分获得情况，以图表形式展示各年级获奖统计 |
| 创新学分 | 管理员可对学生提交的学分申请进行批准或驳回操作，并可填写审批备注 |
| 技术支持 | 管理员可查看或删除学生发布的帖子（不能修改），也可自行发布公告供学生查看 |
| 指导教师 | 管理员可对指导教师信息进行添加、查看、修改、删除操作 |

### 2.3 非功能需求分析

| 需求类型 | 具体要求 |
|----------|---------|
| 安全性 | 系统需实现用户身份认证与授权，采用 JWT Token 机制；不同角色的访问权限需严格隔离；网关层实现全局鉴权过滤 |
| 可用性 | 系统界面需简洁友好，操作流程直观；支持分页查询、表单验证、操作反馈提示等 |
| 可扩展性 | 采用微服务架构，各服务独立部署、独立扩展，新增功能模块不影响现有服务 |
| 性能 | 数据库查询需建立合理的索引；前端采用懒加载、按需渲染等优化策略 |
| 兼容性 | 前端页面需适配主流浏览器（Chrome、Edge、Firefox），采用响应式布局 |

### 3.2 微服务模块划分

项目采用 Maven 多模块管理，共划分为 7 个模块：

| 模块名称 | 功能描述 |
|----------|---------|
| credit-common | 公共模块，存放各微服务共享的实体类、工具类和统一响应结果 |
| credit-gateway | 网关模块，负责请求路由转发、全局鉴权过滤、跨域处理 |
| credit-auth | 认证模块，负责用户登录认证，生成 JWT Token |
| credit-user | 用户模块，管理用户信息、文件上传、密码修改等 |
| credit-competition | 比赛模块，管理比赛信息的 CRUD 操作 |
| credit-apply | 申领模块，管理学分申领记录的提交、审批、查询及数据统计 |
| credit-community | 社区模块，管理帖子的发布、查看、删除及帖子回复 |

## 第五章 接口设计

### 5.1 接口规范

系统采用 RESTful API 风格设计接口，所有接口统一返回 `Result<T>` 格式：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

**状态码说明**：
- 200：操作成功
- 400：请求参数错误
- 401：未授权/Token 无效
- 403：权限不足
- 500：服务器内部错误

### 5.2 接口分组

| 服务 | 接口前缀 | 主要功能 |
|------|---------|---------|
| 认证服务 | /auth | 登录认证、Token 生成 |
| 用户服务 | /user | 个人信息管理、文件上传、学生数据查询 |
| 比赛服务 | /competition | 比赛 CRUD |
| 申领服务 | /apply | 学分申领、审批、统计 |
| 社区服务 | /community | 帖子发布、回复、查看 |

## 第六章 核心功能模块详解

### 6.1 认证与授权模块

#### 6.1.1 JWT Token 认证机制

系统采用 JWT（JSON Web Token）实现无状态身份认证。认证流程如下：

```
┌─────────┐                    ┌─────────┐                    ┌─────────┐
│  前端   │                    │  网关   │                    │ 认证服务 │
└────┬────┘                    └────┬────┘                    └────┬────┘
     │                              │                              │
     │  POST /auth/login            │                              │
     │─────────────────────────────>│                              │
```

#### 6.1.2 网关全局鉴权过滤器

网关模块中的 `AuthGlobalFilter` 实现了 `GlobalFilter` 和 `Ordered` 接口，对所有经过网关的请求进行拦截和 Token 校验。

**核心代码**：

```java
@Component
@Slf4j
public class AuthGlobalFilter implements GlobalFilter, Ordered {

    // 白名单路径，无需 Token 验证
    private static final List<String> WHITE_LIST = Arrays.asList(
            "/auth/login",
            "/doc.html",
            "/webjars/**",
            "/swagger-resources/**",
            "/v3/api-docs/**",
            "/uploads/**"
    );

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        ServerHttpRequest request = exchange.getRequest();
        String path = request.getURI().getPath();

        // 白名单路径直接放行
        if (isWhitePath(path)) {
            return chain.filter(exchange);
        }

        // 获取 Token
        String token = request.getHeaders().getFirst("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            return unauthorized(exchange);
        }

        token = token.substring(7);
        try {
            // 解析 Token
            Claims claims = JwtUtils.parseToken(token);
            
            // 将用户信息写入请求头，传递至下游服务
            ServerHttpRequest newRequest = request.mutate()
                    .header("X-User-Id", claims.get("userId", String.class))
                    .header("X-Username", claims.get("username", String.class))
                    .header("X-Role", claims.get("role", String.class))
                    .build();
            
            return chain.filter(exchange.mutate().request(newRequest).build());
        } catch (Exception e) {
            log.error("Token 解析失败: {}", e.getMessage());
            return unauthorized(exchange);
        }
    }

    @Override
    public int getOrder() {
        return -100; // 优先级最高
    }

    private boolean isWhitePath(String path) {
        for (String whitePath : WHITE_LIST) {
            if (whitePath.endsWith("/**")) {
                String prefix = whitePath.substring(0, whitePath.length() - 3);
                if (path.startsWith(prefix)) {
                    return true;
                }
            } else if (path.equals(whitePath)) {
                return true;
            }
        }
        return false;
    }

    private Mono<Void> unauthorized(ServerWebExchange exchange) {
        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.UNAUTHORIZED);
        response.getHeaders().setContentType(MediaType.APPLICATION_JSON);
        String body = "{\"code\":401,\"message\":\"未授权或Token无效\"}";
        DataBuffer buffer = response.bufferFactory().wrap(body.getBytes());
        return response.writeWith(Mono.just(buffer));
    }
}
```

```javascript
import { defineStore } from 'pinia';
import { ref } from 'vue';

export const useUserStore = defineStore('user', () => {
  const token = ref(localStorage.getItem('token') || '');
  const userInfo = ref(JSON.parse(localStorage.getItem('userInfo') || '{}'));
  
  const setToken = (newToken) => {
    token.value = newToken;
    localStorage.setItem('token', newToken);
  };
  
  const setUserInfo = (info) => {
    userInfo.value = info;
    localStorage.setItem('userInfo', JSON.stringify(info));
  };
  
  const logout = () => {
    token.value = '';
    userInfo.value = {};
    localStorage.removeItem('token');
    localStorage.removeItem('userInfo');
  };
  
  return { token, userInfo, setToken, setUserInfo, logout };
});
```

### 7.4 请求封装

使用 Axios 进行 HTTP 请求封装，统一处理 Token 添加和响应拦截：

```javascript
import axios from 'axios';
import { useUserStore } from '@/stores/user';
import { ElMessage } from 'element-plus';
import router from '@/router';

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
});

// 请求拦截器
request.interceptors.request.use(
  config => {
    const userStore = useUserStore();
    if (userStore.token) {
      config.headers['Authorization'] = `Bearer ${userStore.token}`;
    }
    return config;
  },
  error => Promise.reject(error)
);

// 响应拦截器
request.interceptors.response.use(
  response => {
    const res = response.data;
    if (res.code === 200) {
      return res;
    } else if (res.code === 401) {
      ElMessage.error('登录已过期，请重新登录');
      const userStore = useUserStore();
      userStore.logout();
      router.push('/login');
      return Promise.reject(new Error(res.message));
    } else {
      ElMessage.error(res.message || '请求失败');
      return Promise.reject(new Error(res.message));
    }
  },
  error => {
    ElMessage.error(error.message || '网络错误');
    return Promise.reject(error);
  }
);

export default request;
```

```javascript
// 路由守卫
router.beforeEach((to, from, next) => {
  const userStore = useUserStore();
  
  if (to.meta.requiresAuth && !userStore.token) {
    next('/login');
  } else if (to.meta.roles && !to.meta.roles.includes(userStore.userInfo.role)) {
    ElMessage.error('无权访问该页面');
    next('/');

    next();
```

2. **网关统一鉴权**：通过 Spring Cloud Gateway 实现全局鉴权过滤，简化各服务的认证逻辑。
3. **JWT 无状态认证**：采用 JWT Token 实现无状态身份认证，便于分布式部署。
4. **数据可视化**：使用 ECharts 实现数据统计可视化，直观展示各年级获奖统计、申领状态分布等。
5. **前后端分离**：前端采用 Vue 3 + Vite，后端采用 Spring Boot，通过 RESTful API 进行通信，实现前后端解耦。

### 10.3 未来展望

1. **消息推送**：引入消息队列（如 RabbitMQ），实现审批结果的实时推送通知。
2. **移动端适配**：开发移动端应用或小程序，方便学生随时随地提交申请。
3. **智能推荐**：根据学生的专业和获奖记录，推荐适合的比赛和项目。
4. **数据分析**：引入大数据分析技术，对学生的获奖趋势、学分积累进行深度分析。
5. **流程引擎**：引入工作流引擎（如 Activiti），实现更灵活的审批流程配置。
